# VidGnost 任务处理页性能基线与验收记录

更新时间：2026-04-15

## 1. 文档目标

本文档固定任务处理页的性能观测方法、基线样本和验收阈值，避免后续继续依赖主观体验判断。

## 2. 固定测试场景

### 场景 A：长视频任务详情加载

- 环境：浏览器开发态、Electron 开发态
- 任务样本：长视频 `notes` 任务
- 关注点：
  - 页面是否稳定打开
  - 双栏布局是否完整挂载
  - 转写、笔记、导图、工件区是否同时可用

### 场景 B：长转写列表滚动

- 环境：浏览器开发态、Electron 开发态、Electron 打包态
- 关注点：
  - 转写片段滚动是否持续顺畅
  - 长列表是否持续保持虚拟化
  - 非当前面板刷新是否牵连整页抖动

### 场景 C：长 Markdown / Mermaid 阅读与编辑

- 环境：浏览器开发态、Electron 开发态、Electron 打包态
- 关注点：
  - 任务笔记打开后是否稳定显示
  - 编辑弹窗输入是否持续响应
  - Mermaid 是否在进入可视区后再触发渲染

### 场景 D：运行期高频流式刷新

- 环境：浏览器开发态、Electron 开发态、Electron 打包态
- 关注点：
  - 流式转写期间面板切换是否稳定
  - 阶段动态与问答消息是否采用低阻塞刷新
  - 分栏拖动是否仍可顺滑调整

## 3. 采样方法

### 3.1 构建基线

```powershell
pnpm typecheck
pnpm build
```

记录项：

- 主入口 JS 体积
- CSS 体积
- Worker 产物体积
- 是否存在超大 chunk 告警

### 3.2 浏览器开发态基线

执行方式：

1. 启动后端：`pnpm --filter @vidgnost/api dev`
2. 启动前端：`pnpm --filter @vidgnost/desktop dev --host 127.0.0.1 --port 6221`
3. 打开一个长视频任务详情页
4. 记录：

- 页面是否正常打开
- DOM 节点总数
- 可滚动区域数量
- 虚拟列表当前挂载条目数量
- 控制台是否出现持续重渲染或 store 快照错误

### 3.3 Electron 开发态基线

```powershell
pnpm --filter @vidgnost/desktop desktop:dev
```

录制动作：

- 转写列表滚动 5 秒
- 左右分栏拖动 5 秒
- 打开 Markdown 编辑弹窗并输入 10 秒

记录项：

- Long Task 数量
- 最大单次 Main Thread Task
- FPS 下降区间
- 内存曲线是否持续抬升

### 3.4 Electron 打包态基线

```powershell
pnpm build
```

在打包后的桌面应用中重复 Electron 开发态动作，并记录相同指标。验收以打包态为准。

## 4. 验收阈值

### 浏览器开发态

- 打开任务处理页时不得出现 React 死循环或渲染层崩溃
- 长列表必须保持虚拟化
- Markdown 工作区与转写列表同时存在时页面仍应可滚动、可切换、可点击

### Electron 开发态

- 转写滚动与分栏拖动期间不得出现明显冻结
- 非当前面板更新不得牵连整页同步抖动
- Markdown 编辑器输入不得出现明显字符堆积
- 连续运行 30 分钟后内存曲线不应单向无界上涨

### Electron 打包态

- 打包态体验不得弱于开发态
- 任务处理页打开、滚动、编辑、拖动分栏四类动作都应稳定可用

## 5. 当前观察结论

- 高频运行态数据已经收敛到 task-scoped store
- SSE 事件需要保持批量刷新，避免逐条直接推入重渲染热点
- 长列表必须继续使用虚拟化策略
- Markdown 预处理与 Mermaid 渲染应继续保留 Worker / 可视区惰性策略
- 后续性能评估统一以 TS 后端和 Electron 路径为基准
